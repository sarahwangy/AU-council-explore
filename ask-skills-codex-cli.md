# Codex + Superpowers 使用模板

下面这些提示词可以直接复制到 Codex CLI 里用。目标是让 Superpowers 自动介入，先做需求澄清、方案设计、计划拆解，再进入实现。

## 先说明两点

- `superpowers` 不是 shell 命令，所以在 terminal 里输入 `/superpowers` 不会触发它。
- 它是 Codex 的**插件 + skills 套件**，正常用法是安装后直接用自然语言描述任务，让它自动匹配对应 skill。

### 你会怎么用它

- 在 Codex CLI 里，先用 `/plugins` 查看或确认 `superpowers` 是否已安装。
- 在 Codex App / VS Code extension 里，也是直接用自然语言下任务，不是输入 `/superpowers`。
- 你可以用下面这些短句来主动触发它：
  - `先别写代码，先帮我梳理需求`
  - `先出方案，再写计划`
  - `按 TDD 做`
  - `先做 review，再改`

### 安装范围

- 现在这个环境里的 `superpowers` 是**用户级全局**启用的，不是只在某个项目里。
- 以后别的 plugin 也不一定全局，取决于它是装进 `~/.codex/config.toml` 这样的用户配置，还是某个项目自己的配置。

## 1) 先梳理需求

```text
先不要写代码。

请先帮我梳理这个需求，输出：
1. 你理解的目标
2. 关键约束
3. 需要我确认的问题
4. 可能的实现路径

等我确认后再继续。
```

## 2) 先出设计方案

```text
先做设计，不要直接改代码。

请基于当前仓库和需求，输出一个可执行方案：
1. 现状分析
2. 推荐方案
3. 备选方案
4. 风险和取舍
5. 最终建议

请写得适合我直接审阅。
```

## 3) 先写计划

```text
先不要动手实现。

请把这件事拆成一个清晰的 implementation plan：
1. 按依赖顺序拆步骤
2. 每步都写清文件路径和要改什么
3. 每步都包含验证方式
4. 优先最小改动

等我确认计划后再开始。
```

## 4) 开始执行

```text
计划我确认了。

请按计划执行，优先使用：
- 小步提交思路
- TDD / 先验证再扩展
- 必要时用 subagent 分工

每完成一个阶段都先汇报结果，再继续下一步。
```

## 5) 做前端 / UI 设计

```text
这是一个前端 / UI 任务。

请先从设计角度帮我看：
1. 视觉层级
2. 间距和布局
3. 信息密度
4. 文案是否清晰
5. 哪些地方适合先做低风险优化

先给出设计建议，不要急着改代码。
```

## 6) 修 bug

```text
这是一个 bug 修复任务。

请先定位根因，再修复：
1. 复现路径
2. 可能的根因
3. 你会先检查哪些文件
4. 最小修复方案
5. 如何验证修复有效

不要顺手重构无关部分。
```

## 7) 需要你直接接管流程时

```text
请按 superpowers 的方式处理这件事：
- 先澄清需求
- 再出方案
- 再写计划
- 再执行
- 每一步先确认再推进

如果适用，请优先使用 brainstorming、writing-plans、test-driven-development、subagent-driven-development。
```

## 8) 适合长期固定使用的一句

```text
先别写代码，先按 superpowers 的流程来：澄清需求 → 出方案 → 拆计划 → 再执行。
```
